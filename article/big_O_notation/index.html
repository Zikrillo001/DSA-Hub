Big O Notation

Big O notatsiyaning ta'rifi

Big O notatsiyaning ma'nosi: Big O notatsiya "Algoritmning samaradorligini, ya'ni uning bajarilish vaqtini yoki xotira talablarini, kiritiladigan ma'lumotlarning hajmiga bog'liq holda o'lchash uchun ishlatiladigan matematik ifoda" degan ma'noni anglatadi. U asosan eng yomon holatdagi yoki asimptotik bahoni aniqlash uchun qo'llaniladi.

Illustration

Big O notation is a powerful tool used in computer science to describe the time complexity or space complexity of algorithms. It provides a standardized way to compare the efficiency of different algorithms in terms of their worst-case performance. Understanding Big O notation is essential for analyzing and designing efficient algorithms.

In this tutorial, we will cover the basics of Big O notation, its significance, and how to analyze the complexity of algorithms using Big O.

Big-O Notatsiya nima?

Big-O, odatda “Tartib” deb ataladi, algoritmning vaqt murakkabligi uchun yuqori chegarani ifodalash usuli hisoblanadi, chunki u algoritmning eng yomon holatini tahlil qiladi. Bu algoritmning kiritilgan ma'lumot hajmiga nisbatan qancha vaqt talab qilishini yuqori chegarasini taqdim etadi. U quyidagicha belgilanadi: O(f(n)), bu yerda f(n) algoritmning hajmi n bo'lgan muammoni hal qilish uchun bajaradigan operatsiyalari (qadamlar) sonini ifodalaydi.

Big-O notatsiya algoritmning samaradorligi yoki murakkabligini ifodalash uchun ishlatiladi. Xususan, u eng yomon holatni vaqt yoki xotira murakkabligi jihatidan tasvirlaydi.

Muqim nuqta:

  • Big O notatsiya funksiyaning aniq qiymatini emas, uning asimptotik xatti-harakatini tasvirlaydi.
  • Big O notatsiya turli algoritmlar yoki ma'lumot tuzilmalarining samaradorligini taqqoslash uchun ishlatilishi mumkin.

Big-O Notatsiyaning Ta'rifi:

Ikki funksiya berilgan bo'lsa, f(n) va g(n), biz shunday deymizki, f(n)  O(g(n)) agar shunday  c > 0 va n0 >= 0 bo'lsa, shundayki, f(n) <= c*g(n) hamma uchun n >= n0.

Soddaroq qilib aytganda, f(n) O(g(n)) bo'ladi, agar  f(n) c*g(n) dan tez o'smasa, har bir n >= n0 uchun, bu yerda c va n0 o'zgarmas qiymatlar.

Nega Big-O Notatsiyasi Muhim?

Big-O notatsiyasi algoritmning eng yomon holatdagi vaqt murakkabligini yoki samaradorligini va ma'lumot tuzilmasining eng yomon holatdagi joy murakkabligini tasvirlash uchun ishlatiladigan matematik notatsiyadir. U turli algoritmlar va ma'lumot tuzilmalari ishlashini taqqoslash hamda ularning kiritilgan ma'lumot hajmi oshgani sari qanday o'zgarishini oldindan aytishga imkon beradi.

Big-O notatsiyasi bir nechta sabablar tufayli muhimdir:

  • Big-O notatsiyasi algoritmlarning samaradorligini tahlil qilishga yordam beradi.
  • U algoritmning ish vaqti yoki joy talablari ma'lumot hajmi oshgani sari qanday o'sishini tasvirlash imkonini beradi.
  • Dasturchilarga turli algoritmlarni taqqoslash va muayyan muammo uchun eng samarali algoritmni tanlashga yordam beradi.
  • Algoritmlarning kengayuvchanligini tushunishga va ma'lumot hajmi oshganida ularning ishlashini oldindan aytishga yordam beradi.
  • Kodning optimallashtirilishini ta'minlaydi va umumiy samaradorlikni oshiradi.

Big-O Notatsiyasining Xususiyatlari:

Quyida Big-O notatsiyasining ba'zi muhim xususiyatlari keltirilgan:

1. Refleksivlik:

Har qanday f(n) funksiyasi uchun, f(n) = O(f(n)).

Misol:

f(n) = n2, u holda f(n) = O(n2).

2. Tranzitivlik:

Agar f(n) = O(g(n)) va g(n) = O(h(n)) bo'lsa, u holda f(n) = O(h(n)).

Misol:

f(n) = n3, g(n) = n2, h(n) = n4. U holda f(n) = O(g(n)) va g(n) = O(h(n)). Shunday qilib, f(n) = O(h(n)).

3. Doimiy Ko'paytuvchi:

Har qanday c > 0 doimiy qiymat uchun va f(n) va g(n) funksiyalari uchun, agar f(n) = O(g(n)) bo'lsa, unda cf(n) = O(g(n)).

Misol:

f(n) = n, g(n) = n2. Shunday qilib, f(n) = O(g(n)). Demak, 2f(n) = O(g(n)).

4. Qo'shish Qoidasi:

Agar f(n) = O(g(n)) va h(n) = O(g(n)) bo'lsa, unda f(n) + h(n) = O(g(n)).

Misol:

f(n) = n2, g(n) = n3, h(n) = n4. Shunday qilib, f(n) = O(g(n)) va h(n) = O(g(n)). Demak, f(n) + h(n) = O(g(n)).

5. Ko'paytirish Qoidasi:

Agar f(n) = O(g(n)) va h(n) = O(k(n)) bo'lsa, unda f(n) * h(n) = O(g(n) * k(n)).

Misol:

f(n) = n, g(n) = n2, h(n) = n3, k(n) = n4. Shunday qilib, f(n) = O(g(n)) va h(n) = O(k(n)). Demak, f(n) * h(n) = O(g(n) * k(n)) = O(n6).

6. Tarkib Qoidasi:

Agar f(n) = O(g(n)) va g(n) = O(h(n)) bo'lsa, unda f(g(n)) = O(h(n)).

Misol:

f(n) = n2, g(n) = n, h(n) = n3. Shunday qilib, f(n) = O(g(n)) va g(n) = O(h(n)). Demak, f(g(n)) = O(h(n)) = O(n3).

Keng Tarqalgan Big-O Notatsiyalari:

Big-O notatsiyasi algoritmning vaqt va xotira murakkabligini o'lchash usulidir. U eng yomon holatdagi murakkablikning yuqori chegarasini tasvirlaydi. Keling, turli vaqt murakkabliklariga qaraylik:

1. Lineer Vaqt Murakkabligi: Big O(n) Murakkabligi

Lineer vaqt murakkabligi degani, algoritmning ishlash vaqti kirish hajmi bilan lineer ravishda o'sadi.

Misol uchun, bir algoritmni ko'rib chiqing, u massiv bo'ylab o'ziga xos elementni topish uchun o'tadi:

function findElement(arr, key) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === key) {
            return true;
        }
    }
    return false;
}

2. Logarifmik Vaqt Murakkabligi: Big O(log n) Murakkablik

Logarifmik vaqt murakkabligi algoritmning ish vaqti kiritilgan ma'lumot hajmining logarifmiga mutanosib ekanligini anglatadi.

Masalan, ikkilik qidiruv algoritmi logarifmik vaqt murakkabligiga ega:

function binarySearch(arr, l, r, x) {
    if (r >= l) {
        let mid = Math.floor(l + (r - l) / 2);
        if (arr[mid] === x)
            return mid;
        if (arr[mid] > x)
            return binarySearch(arr, l, mid - 1, x);
        return binarySearch(arr, mid + 1, r, x);
    }
    return -1;
}

3. Kvadratik Vaqt Murakkabligi: Big O(n2) Murakkablik

Kvadratik vaqt murakkabligi algoritmning ish vaqti kiritilgan ma'lumot hajmining kvadratiga mutanosib ekanligini anglatadi.

Masalan, oddiy bubble sort algoritmi kvadratik vaqt murakkabligiga ega:

function bubbleSort(arr) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        for (let j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap function
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
            }
        }
    }
}

4. Kubik Vaqt Murakkabligi: Big O(n3) Murakkablik

Kubik vaqt murakkabligi algoritmning ish vaqti kiritilgan ma'lumot hajmining kubiga mutanosib ekanligini anglatadi.

Masalan, oddiy matritsa ko'paytirish algoritmi kubik vaqt murakkabligiga ega:

function multiply(mat1, mat2) {
    const N = mat1.length;
    let res = new Array(N).fill(0).map(() => new Array(N).fill(0));

    for (let i = 0; i < N; i++) {
        for (let j = 0; j < N; j++) {
            res[i][j] = 0;
            for (let k = 0; k < N; k++) {
                res[i][j] += mat1[i][k] * mat2[k][j];
            }
        }
    }

    return res;
}

5. Ko'phadli Vaqt Murakkabligi: Big O(nk) Murakkablik

Ko'phadli vaqt murakkabligi algoritmning ish vaqtini kiritilgan ma'lumot hajmining ko'phadli funktsiyasi sifatida ifodalash mumkinligini anglatadi n. Big O notatsiyasida, algoritm ko'phadli vaqt murakkabligiga ega deb aytiladi, agar uning vaqt murakkabligi O(nk) bo'lsa, bu yerda k doimiy bo'lib, ko'phadning darajasini ifodalaydi.

Ko'phadli vaqt murakkabligiga ega algoritmlar odatda samarali deb hisoblanadi, chunki kiritilgan ma'lumot hajmi oshgani sari ish vaqti mo'tadil ravishda oshadi. Ko'phadli vaqt murakkabligiga ega algoritmlarning keng tarqalgan misollariga chiziqli vaqt murakkabligi O(n), kvadratik vaqt murakkabligi O(n2) va kubik vaqt murakkabligi O(n3) kiradi.

6. Eksponential Vaqt Murakkabligi: Big O(2n) Murakkabligi

Eksponential vaqt murakkabligi degani, algoritmning ishlash vaqti har bir qo'shilgan ma'lumot to'plami bilan ikki baravar oshadi.

Misol uchun, to'plamning barcha qismlarini yaratish masalasi eksponential vaqt murakkabligiga ega:

function generateSubsets(arr) {
    const n = arr.length;
    for (let i = 0; i < (1 << n); i++) {
        let subset = [];
        for (let j = 0; j < n; j++) {
            if (i & (1 << j)) {
                subset.push(arr[j]);
            }
        }
        console.log(subset.join(" "));
    }
}

Faktorial Vaqt Murakkabligi: Big O(n!) Murakkablik

Faktorial vaqt murakkabligi algoritmning ish vaqti kiritilgan ma'lumot hajmi o'sishi bilan faktorial tarzda ortishini anglatadi. Bu odatda ma'lumotlar to'plamining barcha permutatsiyalarini yaratadigan algoritmlarda kuzatiladi.

Quyida massivning barcha permutatsiyalarini yaratadigan faktorial vaqt murakkabligiga ega algoritmga misol keltirilgan:

function permute(a, l, r) {
    if (l === r) {
        console.log(a.join(" "));
    } else {
        for (let i = l; i <= r; i++) {
            [a[l], a[i]] = [a[i], a[l]];
            permute(a, l + 1, r);
            [a[l], a[i]] = [a[i], a[l]]; // backtrack
        }
    }
}

Big O Notatsiyasini Qanday Aniqlash Kerak?

Big O notatsiyasi bu matematik notatsiya bo'lib, funksiyaning asintotik xatti-harakatlarini uning kiritilishi cheksiz katta bo'lganda tasvirlash uchun ishlatiladi. Bu algoritmlar va ma'lumotlar tuzilmalarining samaradorligini xarakterlash uchun bir yo'ldir.

Big O Notatsiyasini Aniqlash Bosqichlari:

1. Hukmron Atamani Aniqlash:

  • Funksiyani ko'rib chiqing va kirish hajmi ortib borishi bilan eng yuqori o'sish tartibiga ega bo'lgan atamani aniqlang.
  • Hech qanday doimiy faktorlarni yoki pastroq tartibdagi atamalarni e'tiborga olmang.

2. O'sish Tartibini Aniqlash:

  • Hukmron atamaning o'sish tartibi Big O notatsiyasini belgilaydi.

3. Big O Notatsiyasini Yozish:

  • Big O notatsiyasi O(f(n)) tarzida yoziladi, bu yerda f(n) hukmron atamani bildiradi.
  • Masalan, agar hukmron atama n^2 bo'lsa, Big O notatsiyasi O(n^2) bo'ladi.

4. Notatsiyani Soddalashtirish (Ixtiyoriy):

  • Ba'zi holatlarda, Big O notatsiyasini doimiy faktorlardan xalos qilib yoki qisqaroq notatsiyadan foydalanib soddalashtirish mumkin.
  • Misol uchun, O(2n) ni O(n) ga soddalashtirish mumkin.

Misol:

Funksiya: f(n) = 3n3 + 2n2 + 5n + 1

  1. Hukmron Atama: 3n3
  2. O'sish Tartibi: Kubik (n3)
  3. Big O Notatsiyasi: O(n3)
  4. Soddalashtirilgan Notatsiya: O(n3)

Big O Notatsiyasi, Big Ω (Omega) Notatsiyasi va Big θ (Theta) Notatsiyasining Taqqoslanishi:

Quyida Big O notatsiyasi, Ω (Omega) notatsiyasi va θ (Theta) notatsiyasining taqqoslanishi keltirilgan:

NotatsiyaTa'rifIzoh
Big O (O)f(n) ≤ C * g(n) barcha n ≥ n0Algoritmning ish vaqti yuqori chegarasini ta'riflaydi eng yomon holatda.
Ω (Omega)f(n) ≥ C * g(n) barcha n ≥ n0Algoritmning ish vaqti pastki chegarasini ta'riflaydi eng yaxshi holatda.
θ (Theta)C1 * g(n) ≤ f(n) ≤ C2 * g(n) barcha n ≥ n0Algoritmning ish vaqtining yuqori va pastki chegaralarini ta'riflaydi ish vaqti.

Har bir notatsiyada:

  • f(n) tahlil qilinayotgan funktsiyani, odatda algoritmning vaqt murakkabligini anglatadi.
  • g(n) f(n) ni chegaralovchi maxsus funktsiyani anglatadi.
  • C, C1​, va C2​ doimiylar.
  • n0​ tenglikning to'g'ri bo'lishi uchun minimum kirish hajmini anglatadi.

Ushbu notatsiyalar algoritmlarni tahlil qilishda ularning eng yomon holati (Big O), eng yaxshi holati (Ω), va o'rtacha holati (θ) ni ifodalaydi.
