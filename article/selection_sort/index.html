Selection Sort

Algoritmning ta'rifi

Selection Sort algoritmining ta'rifi: Selection Sort — bu taqqoslashga asoslangan saralash algoritmi. U massivni tartiblash uchun har safar saralanmagan qismdan eng kichik (yoki eng katta) elementni tanlab olib, uni saralanmagan qismining birinchi elementi bilan almashtirish orqali ishlaydi. Bu jarayon butun massiv tartiblangan holga kelgunga qadar davom etadi.

Illustration

  1. Dastlab, eng kichik elementni topamiz va uni birinchi element bilan almashtiramiz. Shu bilan eng kichik element o'zining to'g'ri o'rniga joylashtiriladi.
  2. Keyin qolgan elementlar ichida eng kichik (yoki ikkinchi eng kichik) elementni topamiz va uni ikkinchi element bilan almashtiramiz.
  3. Buni davom ettirib, barcha elementlarni o'zining to'g'ri joyiga o'tkazamiz.
function selectionSort(arr) {
    let n = arr.length;
    for (let i = 0; i < n - 1; i++) {
    
        // Assume the current position holds
        // the minimum element
        let min_idx = i;
        
        // Iterate through the unsorted portion
        // to find the actual minimum
        for (let j = i + 1; j < n; j++) {
            if (arr[j] < arr[min_idx]) {
            
                // Update min_idx if a smaller element is found
                min_idx = j;
            }
        }
        
        // Move minimum element to its
        // correct position
        let temp = arr[i];
        arr[i] = arr[min_idx];
        arr[min_idx] = temp;
    }
}

function printArray(arr) {
    for (let val of arr) {
        process.stdout.write(val + " ");
    }
    console.log();
}

// Driver function 
const arr = [64, 25, 12, 22, 11];

console.log("Original array: ");
printArray(arr);

selectionSort(arr);

console.log("Sorted array: ");
printArray(arr);

Selection Sort algoritmining murakkablik tahlili

Vaqt murakkabligi: O(n2), chunki bu yerda ikkita ichki sikl mavjud:

  • Massiv elementlarini birma-bir tanlash uchun birinchi sikl = O(n)
  • Ushbu elementni boshqa har bir massiv elementi bilan solishtirish uchun ikkinchi sikl = O(n)
  • Shunday qilib, umumiy murakkablik = O(n) * O(n) = O(n*n) = O(n2)

Qo‘shimcha xotira: O(1), chunki faqat vaqtinchalik o‘zgaruvchilar uchun qo‘shimcha xotira ishlatiladi.

Selection Sort algoritmining afzalliklari

  • Oddiy tushunish va amalga oshirish oson bo‘lib, asosiy saralash tushunchalarini o‘rgatish uchun ideal.
  • Faqat O(1) qo‘shimcha xotira talab qiladi.
  • Boshqa ko‘plab standart algoritmlarga qaraganda kamroq almashishlar (yoki xotira yozuvlari) talab qiladi. Faqat Cycle Sort xotira yozuvlari bo‘yicha undan ustun keladi. Shu sababli, xotira yozish operatsiyalari qimmat bo‘lgan hollarda oddiy algoritm sifatida tanlanishi mumkin.

Selection Sort algoritmining kamchiliklari

  • Selection Sort algoritmining vaqt murakkabligi O(n^2) bo‘lib, bu uni Quick Sort yoki Merge Sort kabi algoritmlarga nisbatan sekin qiladi.
  • Teng elementlarning nisbiy tartibini saqlamaydi, ya'ni bu algoritm barqaror emas.

Selection Sort algoritmining qo‘llanilish sohalari

  • Asosiy tartiblash mexanizmlari va algoritm tuzishni o‘rgatish uchun juda mos.
  • Kichik ro‘yxatlar uchun mos, chunki murakkabroq algoritmlar uchun ortiqcha resurslar talab qilinmaydi va xotiraga yozish amallari qimmat bo‘lgan holatlarda foydalidir, chunki bu algoritm boshqa standart tartiblash algoritmlariga qaraganda kamroq xotira yozuvlarini talab qiladi.
  • Heap Sort algoritmi Selection Sort asosida yaratilgan.

Selection Sort Code Example

function selectionSort(arr) {
    let n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        let min_index = i;
        for (let j = i + 1; j < n; j++) {
            if (arr[min_index] > arr[j]) {
            min_index = j;
        }
        }
        if (min_index !== i) {
            let a = arr[i];
            arr[i] = arr[min_index];
            arr[min_index] = a;
        }
    }
    return arr;
}
//
//O(1) - xotira
//(n-1)*(n-1) = n^2
// Foydalanish
let array = [64, 25, 12, 22, 11];//4
